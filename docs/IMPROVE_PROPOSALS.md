## A. embedding 영속화(`to_record`) — 직렬화 스키마 설계결정 (P0-3 후속)

**배경**: `infer()` 의 embedding[768] 노출은 적용 완료(IMPROVE_SUMMARY 참조). 그러나
로깅 어댑터 `to_record()` 가 만드는 §2 레코드에 embedding 을 넣는 것은 보류했다.
이유: 레코드의 다른 필드는 전부 스칼라(`float`/`bool`)인데 embedding 은 `(768,)` 배열이라,
parquet/CSV/JSON 직렬화 스키마(스칼라 가정)를 깰 수 있고, 이 환경(torch·pandas-parquet 미설치)에서
**직렬화 경로를 실행검증할 수 없다**. 적대적 리뷰도 "소비자 확인 후 머지" 조건부 판정.

**옵션**
| 옵션 | 방식 | 장점 | 단점 |
|---|---|---|---|
| A1. 인라인 컬럼 | rec 에 `embedding`(768 float32 배열) 직접 포함 | 단일 레코드 완결, P2 소비 단순 | parquet list-column·행당 +~3KB, 스칼라-only 가정 깨짐 |
| A2. 별도 store | raw(II/V2)처럼 embedding 은 별도 parquet/npy 로 분리(`master_clock_ms` 키 조인) | 메인 로그 경량·스칼라 유지 | 조인 로직·동기화 책임 증가 |
| A3. 플래그 게이팅 | `to_record(..., include_embedding=False)` 기본 off, 필요 시 on | 기존 동작 불변, opt-in | 계약에 분기 추가 |

**권장(미확정)**: 메인 로그의 스칼라 스키마를 지키는 **A2(별도 store)** 또는 기존 `include_raw`
패턴과 일관된 **A3**. P2 융합 소비 형태(실시간 dict 직접 소비인지, 로그 재독인지)에 따라 결정.
주의: P2가 `infer()` 반환 dict 를 **실시간 직접 소비**한다면 `to_record` 영속화는 불필요할 수 있음.

## B. INCART 도메인 일반화 역전 — 배포 경로 라우팅 (HANDOFF P0-1)

**사실**: 멀티태스크 5f 의 INCART(병원 Holter) 이진 AUROC=0.284(역전), 동일 조건 이진 단독 ③=0.710
(`records/03` §⑪). 스케일 가설 기각.

**제안(설계결정 — 라우팅 정책)**: 병원 Holter류 도메인에서는 **이진 ③ 경로 사용**을 검토.
이는 추론 라우팅 정책(어느 모델을 언제 쓰는가)이라 코드 자동변경 대상이 아님.
옵션: (B1) 도메인 태그 기반 모델 선택, (B2) 멀티태스크 출력에 INCART형 분포 신뢰도 게이팅,
(B3) 멀티태스크 단일 유지 + 한계 명시 후 실세계 재검증. 트레이드오프: ③은 type 출력 없음(이진만).

## C. CACHET 절대 성능 낙관성 (HANDOFF P0-2)

**사실**: CACHET AUROC(③ 0.844, 5f 0.99대)는 **윈도우 단위·피험자 분리 없음** → 절대값 낙관적.
상대 비교(모델 간)는 유효(`records/03` §⑦·⑪).

**제안**: 코드 변경 아님 — **실세계 종단(피험자 분리) 연구로 절대 성능 재검증** 필수.
배포 문서에 "CACHET 절대 AUROC는 낙관적 상한" 캐비엇 명시(이미 records/05 에 있음).

## D. LoRA 주입 정준 경로 확정 (HANDOFF P1-2)

**사실**: 런타임 monkeypatch(`_inject`/`inject_lora`)와 라이브러리 패치(`patches/lora_fairseq_signals.patch`)
공존. 현 확정 체크포인트는 런타임 주입 경로로 보이나 **정준 여부 미검증**(`ARCHITECTURE.md` §3).

**제안(실행검증 필요 — 이 환경 torch 부재로 불가)**: 검증 환경에서
(D1) 런타임 주입만으로 라이브러리 패치 없이 추론이 동일 결과를 내는지 확인 →
(D2) 정준 경로 1개로 확정·문서화, 불필요 경로는 deprecated 표기.
권장: 이식성 높은 **런타임 주입을 정준으로** 채택하고 패치는 "이전/참고용"으로 명시(미검증 가설).

## E. 이소성 표현 천장 ~0.82 (HANDOFF P1-4, 선택)

**사실**: frozen mean-pool 표현의 이소성 AUROC ~0.82 상한(표현 한계). 첫-10초 crop 폐기율 47.6%로 최고
→ 윈도우 선택 편향 가능성 잔존. 이소성=benign 이라 응급 판정 영향은 작음(`records/05`, `records/03` §5e).

**제안(선택)**: 다중윈도우(MIL) 집계 시도(미실행) 또는 P2 타이밍·자이로 융합. 미션 영향 작아 우선순위 낮음.

## F. mixed_temporal 저SNR 비용 (HANDOFF P1-5)

**사실**: 배포 권장 레시피 mixed_temporal 은 clean·외부 동등하나 저SNR(6~−6dB)에서 −0.004~−0.010
하락. 코드 기본값은 재현성 안전을 위해 `single` 유지(`records/03` §⑩).

**제안**: 실세계 검증에서 저SNR 분포 재확인, 필요 시 `single` 폴백. 코드 기본값 변경은 검증 후 결정.

## G. P2 연결 후속 (HANDOFF P2-1~4, 참고)

P2 통합 단계 작업이라 본 개선 범위 밖. 요지만 인용:
- **P2-1**: `rhythm_regularity` 공식 드리프트(명세 `1-clip(sdNN/200ms)` vs 구현 `1-clip(cv·3)`),
  `model_version` 식별자 비일관 → P2 연결 시 공식 계약 1개로 통일.
- **P2-2**: `benign_flag` 가 명세 출력 계약에 없음 → P2 소비 규약 포함 여부 결정.
- **P2-3**: `inference_ms` 미구현 → 필요 시 추론 진입점 계측 추가.
- **P2-4**: 2리드(II+V2) 동시추론 임계 미검증 → Phase-3 실데이터 재검증.
